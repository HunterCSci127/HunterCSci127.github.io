  • HC 5: Neighborhood Spotlight — Image Processing Due Date: October 7

    For this assignment, you will create a spotlight visualization of your neighborhood: a satellite map image in which everything outside a central rectangular region is converted to grayscale, while the colors in the spotlight area are amplified and a black bounding box is drawn around it. You will write a single Python script that loads the image, processes it with array slicing, and saves one output image. The idea is to gray out the whole image, then paint the spotlight back in with amplified color, and finally draw the box.

    Load the image and set up the spotlight. Start by downloading a .png map image of your neighborhood from Google Maps, zoomed to a level where your neighborhood fills the frame. Ask the user for the input filename and the output filename with input(), then load the image using plt.imread(), which returns a NumPy array of shape height × width × 3 with float values in [0, 1] for PNG files: 

    in_file  = input('Enter the name of your image file: ')
out_file = input('Enter the name for your output file: ')
img = plt.imread(in_file)
    Make a writable copy to modify (call it result), and read the image dimensions from img.shape. Then define your spotlight boundaries as fractions of the image dimensions, for example the top and bottom of the spotlight at one-quarter and three-quarters of the height, and similarly for the left and right edges: 
    height = img.shape[0]
width  = img.shape[1]

top    = height // 4
# Define bottom, left, and right in the same way
    Use // (integer division) here, not /, because an array index has to be a whole number.

    Step 1 — grayscale the whole image. A grayscale pixel has the same value in all three color channels. That value is the pixel's luminance, computed with the weighted formula L = 0.299*R + 0.587*G + 0.114*B. Because the red, green, and blue channels are the three "layers" of the array (img[:, :, 0], img[:, :, 1], and img[:, :, 2]), you can compute the luminance for every pixel at once and then copy it into all three channels of result: 

    L = 0.299 * img[:, :, 0] + 0.587 * img[:, :, 1] + 0.114 * img[:, :, 2]
result[:, :, 0] = L
# Set the other two channels to L as well
    Once all three channels hold L, the entire result image is gray.

    Step 2 — paint the spotlight back in with amplified color. Now think about what luminance actually captures. L is the gray equivalent of a pixel, so channel - L measures how far that channel is from gray, and that deviation is what carries the color. Before writing any code, work through the following:

    • For a pixel (R, G, B) with luminance L, what is R - L? G - L? B - L? What are these values for a pixel that is pure gray?
    • What does L + 0 * (R - L) give for each channel? What about L + 1 * (R - L)? Do the results surprise you?
    • What would L + 2 * (R - L) give?

    To amplify the color, set each channel to L + factor * (channel - L) for the central region only, using a factor greater than 1. Multiplying the deviation by a factor larger than 1 can push a channel value below 0 or above 1, so wrap the result in numpy.clip(array, 0, 1), which caps every value to the valid range (anything below 0 becomes 0, anything above 1 becomes 1). Compute the luminance of the central slice and assign the boosted, clipped values back into that same slice of result: 

    factor = 3.0
center = img[top:bottom, left:right]
Lc = 0.299 * center[:, :, 0] + 0.587 * center[:, :, 1] + 0.114 * center[:, :, 2]

result[top:bottom, left:right, 0] = np.clip(Lc + factor * (center[:, :, 0] - Lc), 0, 1)
# Do the same for channels 1 and 2
    Try factor values of 0, 0.5, 1, 1.5, 2, and 3 and describe what you observe in each case. Which gives the strongest contrast between the spotlight and the gray border? Use a factor of 3 for your final image so the spotlight colors are strongly amplified.

    Step 3 — draw the black bounding box. To make the spotlight stand out, draw a black box along the four edges of the central rectangle. Pick a thickness of a few pixels, then set those edge strips to black by assigning 0 to the first three channels of each strip. The top edge is a strip a few rows tall spanning the width of the spotlight, the left edge is a strip a few columns wide spanning its height, and similarly for the bottom and right: 

    thickness = 4
result[top:top + thickness, left:right, :3] = 0   # top edge
# Now draw the bottom, left, and right edges the same way
    Draw the box last, so it sits on top of the amplified colors.

    Save your result. Save the finished image as a single .png file using the output filename you read earlier: plt.imsave(out_file, result).

  • HC 8: One Block Can Make a Difference Due Date: November 4

    One of the Opportunity Atlas's most striking findings is that neighboring Census tracts can have dramatically different outcomes — children who grew up just one block apart may end up with very different adult incomes. In this assignment, you will look for evidence of this pattern in your borough by pair-programming with your team-mate to develop a Python program.

    To do this:

    • Write a function distance(lat1, lon1, lat2, lon2) that computes the distance in kilometers between two points given their latitude and longitude coordinates, using the Haversine formula (a useful reference: Wikipedia). The Haversine formula, with latitude written as φ, longitude as λ, and the Earth's radius R taken as 6371 km, is

      a = sin²(Δφ/2) + cos φ1 · cos φ2 · sin²(Δλ/2)
      d = 2R · atan2(√a, √(1−a))

      where Δφ is the difference of the two latitudes and Δλ the difference of the two longitudes. The trigonometric functions expect their arguments in radians, so both the latitudes and the longitudes (which the dataset stores in degrees) must be converted first. You can get the math functions you need with from math import radians, sin, cos, sqrt, atan2, where radians(x) converts a value x from degrees to radians. Test your function on a few known pairs of locations (for example, Hunter College at (40.7678, -73.9647) to the Empire State Building at (40.7484, -73.9857) is about 2.8 km).
    • Using your function, identify all pairs of Census tracts in your borough whose centroids are within 1 km of each other (i.e., neighboring tracts). To compare every pair, use a nested loop over the tracts where the inner loop starts one position past the outer one (if the outer index is i, let the inner index j run from i + 1 onward). That way each pair is checked exactly once and no tract is ever compared with itself.
    • For each such pair, compute the absolute difference in kfr_pooled_pooled_p25. Keep in mind that the distance and this difference are two separate quantities: the distance is only used to decide whether a pair counts as neighbors (within 1 km), while the absolute difference in kfr_pooled_pooled_p25 measures how far apart their outcomes are. In other words, you filter pairs by distance, then rank the pairs that survive by their outcome difference.
    • Find the pair with the largest difference. Where are these tracts? What do you know about these neighborhoods from your earlier research?
    • Have your program compute and print a short summary for your borough: the number of tracts analyzed, the number of neighboring pairs (within 1 km), the maximum outcome gap, the number of neighboring pairs with a gap of at least 0.20 and at least 0.30, and the distance between the largest-gap pair. These are the numbers that let you compare your borough to the others in your discussion, so different boroughs will print very different summaries.
    • Create a map that highlights these two tracts and labels each one with its kfr_pooled_pooled_p25 value, so the two outcomes can be compared directly on the map. A natural starting point is the static scatter map you built at the end of HC 6 (each tract drawn as a dot at its centroid and colored by kfr_pooled_pooled_p25): reuse that as your base, then mark and connect your two contrasting tracts on top of it.

    For example, running the program on Manhattan prints a summary like this (if your borough is not Manhattan, your numbers will look quite different): 

    Summary statistics for Manhattan:
  Tracts analyzed:          280
  Neighbor pairs (<= 1 km): 1933
  Max gap:                  0.534
  Pairs with gap >= 0.20:   61
  Pairs with gap >= 0.30:   21
  Top pair distance:        0.81 km

    Program structure requirements:

    • Organize your program into functions. The distance calculation must live in the distance(lat1, lon1, lat2, lon2) function described above, and everything else (loading the data, searching for neighboring pairs, computing the statistics, printing them, and drawing the map) must go inside a main() function. At the very bottom of your program, call main() from inside the standard __main__ guard, so it runs when you run the file directly: 
      if __name__ == "__main__":
    main()
    • Save your map with name hc08_map.png

    Discussion: Write a short paragraph interpreting your finding. Some boroughs are more internally divided than others: in some you will find neighboring tracts whose outcomes differ dramatically, while in others even the largest gap between neighbors is modest and nearby tracts look much alike. Either result is meaningful, so let your data tell you which one your borough has. If you found a striking contrast, what might explain why two tracts so close together diverged so sharply? If instead your borough's neighbors look broadly similar, what does that sameness tell you? In either case, discuss what your result says about your own neighborhood, drawing on what you learned in your earlier research: where do your neighborhood's tracts sit in this picture, and what does that reveal about the place you are studying?

    Note: Your program may take several minutes to run for larger boroughs (Brooklyn and Queens have over 600 tracts each). This is normal — do not interrupt it.

    Extra credit: Speed up your program by adding a bounding box pre-filter. Before computing the full Haversine distance between two tracts, first check whether they are close enough in latitude and longitude to possibly be within 1 km of each other. In NYC (latitude ~40.7°N), 1 km corresponds to roughly 0.009 degrees of latitude and 0.012 degrees of longitude. If two tracts differ by more than these thresholds in either dimension, skip the Haversine calculation entirely. How much faster does your program run with this optimization?

Submit your team's Python program as a .py file to the assignment HC8 - Program, and submit a .pdf containing an image of your map (the hc08_map.png your program saves) and your written interpretation to the assignment HC8 - Discussion.

  • HC 10: Mobility Simulation Due Date: November 18

    The Opportunity Atlas tells us the average adult income rank for children who grew up in each tract — but individual outcomes vary around this average. In this assignment, you will write a simulation that models this variability and uses random numbers to explore what outcomes look like for a large number of hypothetical children from different neighborhoods.

    To do this:

    • Choose three Census tracts from your borough: one with a high mobility score, one with a low score, and one in the middle.
    • For each tract, use random.gauss(mean, std) to simulate the adult income rank of 10,000 hypothetical children, using kfr_pooled_pooled_p25 as the mean and kfr_pooled_pooled_p25_se as the standard deviation. Clip simulated values to the range [0, 1] since income ranks are bounded.
    • Plot the three distributions as overlapping histograms on the same figure, with a different color for each tract.
    • Compute and report: what fraction of simulated children from the low-opportunity tract exceed the mean outcome of the high-opportunity tract?

    Write a short paragraph reflecting on what the simulation reveals. What does overlap between the distributions mean? What would it mean if the distributions did not overlap at all?
